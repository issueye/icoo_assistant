# v0.1.14 开发计划

## 1. 版本定位

`v0.1.14` 计划继续围绕失败复盘体验做小步迭代，把当前已有的失败样本定位提示，再推进到更适合快速比较“同一类失败最近几条样本差异”的方向。

## 2. 版本目标

本版本计划交付以下能力：

- 增强优先失败样本的对比提示
- 优化从样本定位到详细历史的路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.14`

- 异常复盘相关审计增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.14`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 优先提升失败样本对比效率
- 不破坏既有工具边界

### 4.2 版本重点

- 优先失败样本的对比提示
- 失败复盘路径的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### B. 演示整理

### C. 测试与发布

## 7. 完成度评估

### 7.1 当前结论

`v0.1.14` 可以判定为已完成，当前任务侧审计已经能直接对比优先失败原因最近两条样本的关键差异，帮助更快判断问题是在稳定重复还是已经发生变化。

完成依据：

- 已在 `task_audit summary` 中新增 `priority_failure_sample_compare`
- 当前输出会对比最近两条同类失败样本的 `command` 与错误签名是否变化
- README 与 `tool_catalog` 已同步补上失败样本对比提示的示例与导航
- 相关测试已补充并通过 `go test ./...`

### 7.2 当前边界

- 当前比较仍然只覆盖最近两条样本，而不是更长窗口
- 对比维度仍然较轻，只覆盖命令文本和归一化错误签名
- 还没有把差异进一步归纳成更高层的结论或建议

## 8. 风险提示

- 如果对比提示过重，可能影响摘要的快速浏览体验
- 若对比规则不稳定，可能增加理解成本
- 仍需持续控制范围膨胀

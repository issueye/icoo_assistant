# v0.1.18 开发计划

## 1. 版本主题

让 `task_audit history` 在配合双样本对比时，直接标出 latest / previous 角色，减少用户自己对齐顺序的成本。

## 2. 为什么现在做

`v0.1.17` 已经能告诉用户应该优先看哪两条样本，下一步最自然的问题就是进入 `history` 后如何更快识别哪条是最新、哪条是上一条。这一版继续停留在查看体验层，不扩展为自动 diff 或更重的诊断结论。

## 3. 计划变更

- 为 `task_audit action=history` 的最近双样本场景增加 latest / previous 角色提示
- 让 reason 过滤后的最近两条失败样本更容易按顺序阅读
- README 与 `tool_catalog` 同步更新
- 增加对应测试

## 4. 本版不做

- 自动生成样本差异明细
- 多样本批量比较
- 跨任务历史配对

## 5. 验收标准

- [ ] `task_audit history` 在双样本查看场景下新增角色提示
- [ ] README 与 `tool_catalog` 已更新
- [ ] 相关测试补充完成
- [ ] `go test ./...` 通过

## 6. 风险与停手线

- 如果把 role 标记做得太重，会影响 history 的紧凑输出
- 如果 role 提示开始附带过多解释，会与 summary 导航字段重叠
- 因此本版应控制在“帮助识别顺序”而不是“直接输出完整 diff”

# v0.1.25 开发计划

## 1. 版本主题

让 `task_audit history` 在聚焦失败查看时补上一条 latest failure entry hint，减少用户继续在条目列表里人工确认最新失败对应的是哪一行。

## 2. 为什么现在做

`v0.1.24` 已经能让用户更快看到最新失败的时间点，下一步最自然的问题就是“这条最新失败在当前返回结果里具体对应哪一行”。这一版继续停留在 history 可读性层，不扩展到新的推断逻辑。

## 3. 计划变更

- 为 `task_audit history` 增加一个轻量 latest failure entry hint
- 让失败历史视图更容易快速把 latest 提示和具体条目对应起来
- README 与 `tool_catalog` 同步更新
- 增加对应测试

## 4. 本版不做

- 自动高亮 UI
- 更复杂的历史排序策略
- 新的诊断或修复建议

## 5. 验收标准

- [ ] `task_audit history` 新增 latest failure entry hint
- [ ] README 与 `tool_catalog` 已更新
- [ ] 相关测试补充完成
- [ ] `go test ./...` 通过

## 6. 风险与停手线

- 如果 latest failure entry hint 和现有 `latest_sample` 重复过多，会削弱 history 的紧凑性
- 如果提示开始解释为什么这条更重要，会和 summary 的导航字段重叠
- 因此本版应控制在“更快对上条目”而不是“追加新的判断层”

# v0.1.3 开发计划

## 1. 版本定位

`v0.1.3` 计划继续围绕当前单 Agent 演示底座做小步打磨，把审计入口从“知道怎么用”再推进到“更容易快速定位目标信息”。

## 2. 版本目标

本版本计划交付以下能力：

- 增强审计查询的过滤或摘要体验
- 进一步降低首次排障的阅读成本
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.3`

- 审计输出格式或过滤体验增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.3`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 继续保持范围克制
- 优先提升排障和演示体验
- 尽量复用现有工具边界

### 4.2 版本重点

- 审计信息的可定位性
- 输出格式可读性
- 文档与代码一致性

## 5. 任务拆解

### A. 审计体验

- 评估 `task_audit` 和 `agent_hook_audit` 是否需要更细过滤或摘要
- 优化最常见排障场景的信息密度

### B. 演示整理

- 增加更贴近真实排障路径的示例
- 保持 README 简洁但可操作

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [ ] 审计结果更容易定位重点
- [ ] 演示与排障示例更实用
- [ ] 测试通过
- [ ] 文档更新

## 7. 风险提示

- 过滤能力增强后，输出约定需要保持稳定
- 如果摘要过度，可能丢失原始排障线索
- 需要继续避免范围膨胀

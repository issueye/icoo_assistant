# v0.1.9 开发计划

## 1. 版本定位

`v0.1.9` 计划继续围绕失败复盘体验做小步迭代，把当前已有的失败原因聚焦，再推进到更适合快速判断“下一步先排查哪一类问题”的方向。

## 2. 版本目标

本版本计划交付以下能力：

- 增强失败原因的优先级判断体验
- 优化从失败聚焦到下一步排查的路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.9`

- 异常复盘相关审计增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.9`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 优先提升失败排查决策速度
- 不破坏既有工具边界

### 4.2 版本重点

- 失败原因的优先排查提示
- 失败复盘路径的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### A. 异常复盘

- 评估是否需要在失败概况中补充优先排查建议或重点原因提示
- 优化从失败原因聚焦到详细历史查看的路径

### B. 演示整理

- 增加更贴近失败优先级判断的示例
- 保持 README 简洁可执行

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [x] 更容易判断下一步优先排查的失败原因
- [x] 失败复盘路径更顺滑
- [x] 测试通过
- [x] 文档更新

## 7. 完成度评估

### 7.1 当前结论

`v0.1.9` 可以判定为已完成，当前任务侧审计已经能在失败概况里直接给出优先排查的失败原因和下一步建议，减少从概况跳转到详细历史时的判断成本。

完成依据：

- 已在 `task_audit summary` 中新增 `priority_failure_reason` 和 `priority_failure_hint`
- 优先排查原因会综合失败数量，并在并列时优先选择最近仍在出现的失败类型
- README 与 `tool_catalog` 已同步补上优先排查提示的示例与导航说明
- 相关测试已补充并通过 `go test ./...`

### 7.2 当前边界

- 当前优先级仍然基于轻量规则，只考虑失败数量和最近性
- 仍然是单任务视角，暂不支持跨任务失败优先级汇总
- 优先提示还没有给出更细的修复建议模板

## 8. 风险提示

- 如果优先级规则继续扩展，需要避免解释成本上升
- 若建议字段持续增加，可能影响摘要首屏可读性
- 仍需持续控制范围膨胀

# v0.1.19 开发计划

## 1. 版本主题

让 `task_audit history` 在双样本聚焦场景里补上一条轻量 pair 摘要，帮助用户更快理解这两条样本的主要差别。

## 2. 为什么现在做

`v0.1.18` 已经解决了“哪条是 latest、哪条是 previous”的识别问题，下一步最自然的问题就是“这两条最主要差在哪里”。这一版继续停留在查看辅助层，不扩展到完整 diff 或自动诊断。

## 3. 计划变更

- 为 `task_audit action=history` 的聚焦双样本场景增加轻量 pair 摘要
- 让命令变化与错误签名变化在 history 视图里更快可见
- README 与 `tool_catalog` 同步更新
- 增加对应测试

## 4. 本版不做

- 完整字段级 diff
- 三条及以上样本比较
- 自动根因归纳

## 5. 验收标准

- [x] `task_audit history` 在双样本聚焦场景下新增轻量 pair 摘要
- [x] README 与 `tool_catalog` 已更新
- [x] 相关测试补充完成
- [x] `go test ./...` 通过

## 6. 完成度评估

### 6.1 当前结论

`v0.1.19` 可以判定为已完成，当前任务侧审计已经能在 focused history 里补上一条轻量 `pair_summary`，帮助用户在 `previous/latest` 角色之外，进一步快速判断这两条样本的主要差别。

完成依据：

- 已在 `task_audit action=history` 的双样本聚焦场景中新增 `pair_summary`
- 当前会直接概括 `command` 与 `error_signature` 是 `same` 还是 `changed`
- README 与 `tool_catalog` 已同步补上轻量 pair 摘要说明
- 相关测试已补充并通过 `go test ./...`

### 6.2 当前边界

- 当前 `pair_summary` 仍然只覆盖最近两条聚焦样本
- 当前只概括命令与错误签名变化，不进入完整字段级 diff
- 当前不会解释变化原因，只帮助更快阅读 pair

## 7. 风险与停手线

- 如果 pair 摘要承载太多信息，会让 history 重新变重
- 如果 pair 摘要开始解释原因，会和 summary 的变化点字段重叠
- 因此本版应控制在“快速概括差别”而不是“输出详细 diff 结果”

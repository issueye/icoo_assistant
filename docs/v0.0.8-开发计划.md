# v0.0.8 开发计划

## 1. 版本定位

`v0.0.8` 聚焦把当前“任务对象里有有限执行历史”的能力推进到“更适合查询和审计的历史查看体验”。核心目标是让 Agent 能更清楚地查看任务执行轨迹，而不让主任务输出变得过重。

## 2. 版本目标

本版本计划交付以下能力：

- 更清晰的任务历史查看入口
- 历史输出的紧凑化或分页化思路
- 相关测试和文档

## 3. 范围定义

### 3.1 纳入 `v0.0.8`

- `project_task` 历史查看增强
- 紧凑历史输出规则
- 文档与测试

### 3.2 暂不纳入 `v0.0.8`

- 自动任务拆解
- 自动认领任务
- 多 Agent 团队协作
- 审批协议
- worktree 绑定

## 4. 设计重点

### 4.1 核心原则

- 默认输出继续保持紧凑
- 历史查看入口要明确
- 为后续独立审计日志预留空间

### 4.2 版本重点

- 先解决“怎么看历史更顺手”
- 再考虑“历史如何独立存储”

## 5. 任务拆解

### A. 输出增强

- 设计更清晰的历史展示方式
- 考虑动作级查看入口

### B. 测试验证

- 增加历史查看测试
- 保证默认输出不过重

### C. 文档收口

- 更新 README
- 更新版本文档

## 6. 验收清单

- [ ] 任务历史查看体验得到增强
- [ ] 默认输出仍保持紧凑
- [ ] 测试通过
- [ ] 文档更新

## 7. 风险提示

- 如果历史展示过度展开，会显著抬高上下文成本
- 如果入口过多，工具职责会变得混乱
- 输出规则需要兼顾易读性和后续可扩展性

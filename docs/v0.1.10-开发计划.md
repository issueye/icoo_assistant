# v0.1.10 开发计划

## 1. 版本定位

`v0.1.10` 计划继续围绕失败复盘体验做小步迭代，把当前已有的优先排查提示，再推进到更适合快速理解“为什么它现在最值得先看”的方向。

## 2. 版本目标

本版本计划交付以下能力：

- 增强优先排查提示的解释性
- 优化从优先提示到详细复盘的路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.10`

- 异常复盘相关审计增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.10`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 优先提升排障判断的可解释性
- 不破坏既有工具边界

### 4.2 版本重点

- 优先排查提示的解释信息
- 失败复盘路径的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### A. 异常复盘

- 评估是否需要补充优先原因的选择依据或摘要解释
- 优化从优先提示到失败历史查看的路径

### B. 演示整理

- 增加更贴近优先级解释的示例
- 保持 README 简洁可执行

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [ ] 更容易理解优先排查提示的依据
- [ ] 失败复盘路径更顺滑
- [ ] 测试通过
- [ ] 文档更新

## 7. 风险提示

- 如果解释字段过多，可能让摘要重新变重
- 若规则表达不稳定，可能削弱用户信任
- 仍需持续控制范围膨胀

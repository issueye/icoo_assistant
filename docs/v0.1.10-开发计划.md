# v0.1.10 开发计划

## 1. 版本定位

`v0.1.10` 计划继续围绕失败复盘体验做小步迭代，把当前已有的优先排查提示，再推进到更适合快速理解“为什么它现在最值得先看”的方向。

## 2. 版本目标

本版本计划交付以下能力：

- 增强优先排查提示的解释性
- 优化从优先提示到详细复盘的路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.10`

- 异常复盘相关审计增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.10`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 优先提升排障判断的可解释性
- 不破坏既有工具边界

### 4.2 版本重点

- 优先排查提示的解释信息
- 失败复盘路径的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### A. 异常复盘

- 评估是否需要补充优先原因的选择依据或摘要解释
- 优化从优先提示到失败历史查看的路径

### B. 演示整理

- 增加更贴近优先级解释的示例
- 保持 README 简洁可执行

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [x] 更容易理解优先排查提示的依据
- [x] 失败复盘路径更顺滑
- [x] 测试通过
- [x] 文档更新

## 7. 完成度评估

### 7.1 当前结论

`v0.1.10` 可以判定为已完成，当前任务侧审计已经能说明优先排查提示是因为失败次数更多，还是因为在并列时最近仍在出现，从而让失败复盘结论更可解释。

完成依据：

- 已在 `task_audit summary` 中新增 `priority_failure_basis`
- 优先排查依据会明确说明是 `highest_count` 还是 `latest_occurrence`
- README 与 `tool_catalog` 已同步补上优先依据说明的示例与导航
- 相关测试已补充并通过 `go test ./...`

### 7.2 当前边界

- 当前解释仍然基于轻量规则，只覆盖次数和最近性
- 仍然是单任务视角，暂不支持跨任务优先依据汇总
- 还没有把失败命令、错误摘要进一步压缩成更高层解释

## 8. 风险提示

- 如果解释字段继续增加，可能让摘要重新变重
- 若规则表达不稳定，可能削弱用户信任
- 仍需持续控制范围膨胀

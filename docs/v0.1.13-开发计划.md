# v0.1.13 开发计划

## 1. 版本定位

`v0.1.13` 计划继续围绕失败复盘体验做小步迭代，把当前已有的重复模式提示，再推进到更适合快速判断“下一步应该优先看哪条具体失败样本”的方向。

## 2. 版本目标

本版本计划交付以下能力：

- 增强优先失败样本的定位提示
- 优化从重复模式提示到详细历史的路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.13`

- 异常复盘相关审计增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.13`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 优先提升失败样本定位速度
- 不破坏既有工具边界

### 4.2 版本重点

- 优先失败样本的定位提示
- 失败复盘路径的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### A. 异常复盘

- 评估是否需要补充优先失败原因的样本定位提示或推荐查看入口
- 优化从重复模式提示到失败历史查看的路径

### B. 演示整理

- 增加更贴近失败样本定位的示例
- 保持 README 简洁可执行

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [x] 更容易定位下一步优先查看的失败样本
- [x] 失败复盘路径更顺滑
- [x] 测试通过
- [x] 文档更新

## 7. 完成度评估

### 7.1 当前结论

`v0.1.13` 可以判定为已完成，当前任务侧审计已经能直接给出建议优先查看的失败样本定位信息，减少从摘要跳到详细历史时的来回筛选。

完成依据：

- 已在 `task_audit summary` 中新增 `priority_failure_sample_target`
- 当前输出会直接给出建议优先查看的 `job_id` 以及对应的 `task_audit history` 查询入口
- README 与 `tool_catalog` 已同步补上失败样本定位提示的示例与导航
- 相关测试已补充并通过 `go test ./...`

### 7.2 当前边界

- 当前样本定位仍然只围绕优先失败原因的最近样本
- 仍然没有支持按 `job_id` 直接精确查询历史明细
- 还没有把多条候选样本做进一步排序比较

## 8. 风险提示

- 如果定位提示过强，可能掩盖其他同样重要的失败样本
- 若新增摘要过多，可能削弱首屏可读性
- 仍需持续控制范围膨胀

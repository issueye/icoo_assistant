# v0.1.2 开发计划

## 1. 版本定位

`v0.1.2` 计划继续围绕当前可演示运行时做收口，把任务审计、hook 审计和工具目录三条线进一步串成更顺滑的回看路径。

## 2. 版本目标

本版本计划交付以下能力：

- 增强跨审计入口的发现与导航体验
- 补齐更明确的审计查询示例
- 继续打磨文档和测试

## 3. 范围定义

### 3.1 纳入 `v0.1.2`

- 审计相关工具的说明或导航增强
- 更顺滑的演示路径
- 文档与测试

### 3.2 暂不纳入 `v0.1.2`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 完整可视化界面

## 4. 设计重点

### 4.1 核心原则

- 继续优先补强现有底座的可理解性
- 降低首次演示和排障的认知负担
- 保持工具职责边界稳定

### 4.2 版本重点

- 审计入口串联
- 演示文案与示例优化
- 测试与文档一致性

## 5. 任务拆解

### A. 审计入口梳理

- 评估 `tool_catalog / task_audit / agent_hook_audit` 之间的导航关系
- 增强审计入口提示或返回信息

### B. 演示路径整理

- 补充更直接的审计查询示例
- 让 README 更适合首次体验

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [ ] 审计入口关系更清晰
- [ ] 演示路径更顺滑
- [ ] 测试通过
- [ ] 文档更新

## 7. 风险提示

- 如果返回提示过多，可能让工具输出变得啰嗦
- 导航增强需要避免影响现有简洁输出
- 仍需克制范围，避免重新回到大功能堆叠

# v0.1.0 开发计划

## 1. 版本定位

`v0.1.0` 作为当前单 Agent 工程底座的阶段性里程碑，目标是把已经具备的任务、后台执行、埋点和审计查询能力进一步收敛成一个更稳定、更可演示的最小可用运行时。

## 2. 版本目标

本版本计划交付以下能力：

- 对现有核心能力做一次收敛和整理
- 提升工具职责边界与说明清晰度
- 补强最关键的可演示路径和文档

## 3. 范围定义

### 3.1 纳入 `v0.1.0`

- 核心工具和模块职责梳理
- README 与使用说明增强
- 关键路径验证和测试补强

### 3.2 暂不纳入 `v0.1.0`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- worktree 绑定

## 4. 设计重点

### 4.1 核心原则

- 先把已有能力打磨清楚
- 优先提升可理解性和可演示性
- 为后续 `v0.2.x` 多 Agent 方向保留空间

### 4.2 版本重点

- 统一工具职责说明
- 优化仓库文档结构
- 让演示路径更顺滑

## 5. 任务拆解

### A. 能力梳理

- 盘点现有工具和模块职责
- 明确 `todo / project_task / background / task_audit / task` 分工

### B. 体验收口

- 优化 README 和示例
- 补齐关键使用路径说明

### C. 测试与发布

- 复核关键路径测试
- 更新版本文档

## 6. 验收清单

- [x] 核心能力边界说明更清晰
- [x] 文档更适合新开发者上手
- [x] 关键测试通过
- [x] 版本文档更新

## 7. 完成度评估

### 7.1 当前结论

`v0.1.0` 可以判定为已完成，当前单 Agent 运行时已经具备更明确的工具职责说明、更一致的版本标识，以及更适合作为演示入口的文档与示例路径。

完成依据：

- 已新增 `tool_catalog` 工具，支持 `list / describe`
- 已将 `tool_catalog` 接入主运行时和子代理基础工具集合
- 已把 `project_task / task_audit / task / background / bash / todo` 等职责边界收敛到统一说明入口
- CLI `--version` 已与当前版本计划对齐到 `0.1.0`
- README 已补齐演示路径与工具边界说明
- 已补充 `tool_catalog` 与 CLI 帮助相关测试，`go test ./...` 通过

### 7.2 当前边界

- 工具目录仍然是静态描述，还没有根据运行时插件或扩展能力动态生成
- 当前边界说明以文本输出为主，尚未提供机器可消费的结构化目录格式
- hook 数据目前已落盘，但还没有独立查询工具来直接检索 `.agent-hooks/events.jsonl`

## 8. 风险提示

- 如果目录说明和实现演进不同步，会造成说明失真
- 若后续工具数量快速增加，静态 catalog 维护成本会上升
- `v0.1.0` 仍应保持“收口与打磨”为主，避免重新膨胀为功能堆叠

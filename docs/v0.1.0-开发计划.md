# v0.1.0 开发计划

## 1. 版本定位

`v0.1.0` 作为当前单 Agent 工程底座的阶段性里程碑，目标是把已经具备的任务、后台执行、埋点和审计查询能力进一步收敛成一个更稳定、更可演示的最小可用运行时。

## 2. 版本目标

本版本计划交付以下能力：

- 对现有核心能力做一次收敛和整理
- 提升工具职责边界与说明清晰度
- 补强最关键的可演示路径和文档

## 3. 范围定义

### 3.1 纳入 `v0.1.0`

- 核心工具和模块职责梳理
- README 与使用说明增强
- 关键路径验证和测试补强

### 3.2 暂不纳入 `v0.1.0`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- worktree 绑定

## 4. 设计重点

### 4.1 核心原则

- 先把已有能力打磨清楚
- 优先提升可理解性和可演示性
- 为后续 `v0.2.x` 多 Agent 方向保留空间

### 4.2 版本重点

- 统一工具职责说明
- 优化仓库文档结构
- 让演示路径更顺滑

## 5. 任务拆解

### A. 能力梳理

- 盘点现有工具和模块职责
- 明确 `todo / project_task / background / task_audit / task` 分工

### B. 体验收口

- 优化 README 和示例
- 补齐关键使用路径说明

### C. 测试与发布

- 复核关键路径测试
- 更新版本文档

## 6. 验收清单

- [ ] 核心能力边界说明更清晰
- [ ] 文档更适合新开发者上手
- [ ] 关键测试通过
- [ ] 版本文档更新

## 7. 风险提示

- 如果范围扩张过快，会把 `v0.1.0` 重新拉回功能堆叠
- 文档调整需要避免与现有实现脱节
- 需要持续保持版本边界克制

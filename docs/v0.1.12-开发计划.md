# v0.1.12 开发计划

## 1. 版本定位

`v0.1.12` 计划继续围绕失败复盘体验做小步迭代，把当前已有的优先失败样本摘要，再推进到更适合快速判断“这一类失败最近有没有明显重复模式”的方向。

## 2. 版本目标

本版本计划交付以下能力：

- 增强优先失败原因的重复模式提示
- 优化从优先样本摘要到详细历史的路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.12`

- 异常复盘相关审计增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.12`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 优先提升失败模式识别速度
- 不破坏既有工具边界

### 4.2 版本重点

- 优先失败原因的重复模式提示
- 失败复盘路径的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### A. 异常复盘

- 评估是否需要补充优先失败原因的重复错误片段或模式提示
- 优化从优先样本摘要到失败历史查看的路径

### B. 演示整理

- 增加更贴近失败模式识别的示例
- 保持 README 简洁可执行

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [x] 更容易判断优先失败原因是否在重复出现
- [x] 失败复盘路径更顺滑
- [x] 测试通过
- [x] 文档更新

## 7. 完成度评估

### 7.1 当前结论

`v0.1.12` 可以判定为已完成，当前任务侧审计已经能在优先失败原因旁边给出轻量的重复模式提示，帮助更快判断这类失败是偶发样本还是最近在持续复现。

完成依据：

- 已在 `task_audit summary` 中新增 `priority_failure_pattern_hint`
- 重复模式提示会基于优先失败原因最近几条样本，给出 `repeat` 或 `single` 的轻量判断
- README 与 `tool_catalog` 已同步补上重复模式提示的示例与导航
- 相关测试已补充并通过 `go test ./...`

### 7.2 当前边界

- 当前模式提示仍然是轻量规则，只覆盖最近少量样本
- 对超时类错误会做基础归一化，其他错误仍主要按原始文本判断
- 还没有进一步把重复模式映射到修复建议或更高层标签

## 8. 风险提示

- 如果模式提示继续扩展，可能误导实际排障方向
- 若新增摘要过多，可能削弱首屏可读性
- 仍需持续控制范围膨胀

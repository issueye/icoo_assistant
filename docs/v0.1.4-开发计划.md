# v0.1.4 开发计划

## 1. 版本定位

`v0.1.4` 计划继续沿着审计可读性做小步迭代，把当前已经有的摘要、过滤和导航再推进到更实用的“问题定位”体验。

## 2. 版本目标

本版本计划交付以下能力：

- 增强更贴近排障的审计筛选或排序体验
- 优化最常见异常场景的查看路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.4`

- 审计查询体验增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.4`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 继续优先解决排障时的真实痛点
- 不破坏既有工具边界

### 4.2 版本重点

- 异常信息的可定位性
- 常见排障流程的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### A. 审计体验

- 评估是否需要更细粒度的筛选、排序或异常聚焦能力
- 优化失败场景下的信息组织方式

### B. 演示整理

- 增加更贴近异常排障的示例
- 保持 README 简洁可执行

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [ ] 异常排障路径更顺滑
- [ ] 审计查询更容易定位失败信息
- [ ] 测试通过
- [ ] 文档更新

## 7. 风险提示

- 如果筛选维度继续增加，需要避免输出约定变复杂
- 若过度围绕异常场景优化，可能牺牲通用性
- 仍需持续控制范围膨胀

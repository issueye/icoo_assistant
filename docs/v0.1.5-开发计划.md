# v0.1.5 开发计划

## 1. 版本定位

`v0.1.5` 计划继续围绕异常排障体验做小步迭代，把单任务失败概况再推进到更适合快速定位问题根因的方向。

## 2. 版本目标

本版本计划交付以下能力：

- 增强失败信息的聚焦或分类体验
- 优化最常见异常分析路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.5`

- 异常排障相关审计增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.5`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 优先提升真实排障场景的信噪比
- 不破坏既有工具边界

### 4.2 版本重点

- 失败原因的可定位性
- 异常分析路径的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### A. 异常排障

- 评估是否需要更细的失败分类、过滤或提示
- 优化从失败概况到详细信息的下钻路径

### B. 演示整理

- 增加更贴近失败排障的示例
- 保持 README 简洁可执行

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [ ] 失败原因更容易被定位
- [ ] 失败排障路径更顺滑
- [ ] 测试通过
- [ ] 文档更新

## 7. 风险提示

- 如果失败分类设计过早定型，后续扩展成本会提高
- 若异常提示过多，可能让输出重新变长
- 仍需持续控制范围膨胀

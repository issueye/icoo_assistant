## 四、未完成 / 缺失项

### 4.1 测试缺口（3 个文件缺专用测试）

| 文件 | 风险等级 | 说明 |
|------|----------|------|
| `internal/tools/files.go` | 🟡 中 | read/write/edit_file 工具 Handler 无直接单元测试，仅底层 workspace 有测试 |
| `internal/commandutil/exec.go` | 🟡 中 | Shell 执行逻辑无专用测试，仅通过集成测试间接覆盖 |
| `internal/subagent/runner.go` | 🟡 中 | 子智能体运行器无专用测试，仅通过 agent/loop_test.go 间接覆盖 |

### 4.2 安全加固缺口

| 问题 | 严重度 | 建议 |
|------|--------|------|
| 危险命令拦截仅 5 条规则 | 🔴 高 | 需增加：`rm -rf *`、`mkfs`、`dd`、`chmod -R 777 /`、fork bomb、管道注入等 |
| 拦截使用简单子串匹配 | 🟡 中 | 可被 `su\do`、`rm\ -rf\ /` 等方式绕过 |
| 缺少 URL 注入防护 | 🟡 中 | `curl <url> \| sh` 类模式未拦截 |

### 4.3 文档缺失

| 缺失项 | 影响 |
|--------|------|
| 项目根目录缺 `README.md` | 新用户无入门指引 |
| `skills/` 目录不存在 | `load_skill` 工具只能返回 "no skills available" |
| 无 API 文档 | 二次开发者需阅读源码 |

### 4.4 配置问题

| 问题 | 影响 |
|------|------|
| 配置加载后无合法性校验 | `MaxRounds=0` 会被默认值兜底，但 `= -1` 不会 |
| `go.mod` 依赖全部标记为 `// indirect` | 应显式声明 `anthropic-sdk-go` 为直接依赖 |

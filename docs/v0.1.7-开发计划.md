# v0.1.7 开发计划

## 1. 版本定位

`v0.1.7` 计划继续围绕失败复盘体验做小步迭代，把当前已有的失败样本对比再推进到更适合快速追踪变化趋势的方向。

## 2. 版本目标

本版本计划交付以下能力：

- 增强失败样本的趋势或时间维度可读性
- 优化失败复盘路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.7`

- 异常复盘相关审计增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.7`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 优先提升失败复盘时的可读性
- 不破坏既有工具边界

### 4.2 版本重点

- 失败趋势的可观察性
- 失败复盘路径的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### A. 异常复盘

- 评估是否需要时间维度、趋势提示或更细的复盘入口
- 优化从失败样本对比到详细历史的路径

### B. 演示整理

- 增加更贴近失败复盘的示例
- 保持 README 简洁可执行

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [x] 失败趋势更容易被观察
- [x] 失败复盘路径更顺滑
- [x] 测试通过
- [x] 文档更新

## 7. 完成度评估

### 7.1 当前结论

`v0.1.7` 可以判定为已完成，当前任务侧审计已经能在失败概况里给出最近失败趋势，失败复盘时更容易观察到近期变化。

完成依据：

- 已在 `task_audit summary` 中新增 `recent_failure_trend`
- 失败概况现在不仅有分类统计和最近样本，还能看到最近失败的时间顺序
- README 与 `tool_catalog` 已同步补上失败趋势导向的示例与说明
- 相关测试已补充并通过 `go test ./...`

### 7.2 当前边界

- 当前趋势仍然是最近少量失败样本视图，还没有更长窗口的趋势统计
- 趋势仍然基于单任务，不支持跨任务横向观察
- 失败趋势展示偏文本概览，还没有结构化对比接口

## 8. 风险提示

- 如果趋势信息继续增加，需要避免摘要过重
- 若时间维度展示不克制，可能影响快速浏览
- 仍需持续控制范围膨胀

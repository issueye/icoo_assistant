# v0.1.16 开发计划

## 1. 版本主题

让 `task_audit summary` 在给出趋势后，进一步提示最近几条同类失败里最值得对比的变化点。

## 2. 为什么现在做

`v0.1.15` 已经能判断“稳定还是变化”，下一步自然是回答“具体哪里变了”。这一版计划继续留在摘要层，不直接扩展到更复杂的诊断或修复建议。

## 3. 计划变更

- 为优先失败原因增加更聚焦的变化点摘要
- 让最近样本对比信息更容易快速浏览
- README 与 `tool_catalog` 同步更新
- 增加对应测试

## 4. 本版不做

- 跨任务失败聚合
- 自动修复建议
- 可视化趋势或图表

## 5. 验收标准

- [ ] `task_audit summary` 新增一个面向变化点的摘要字段
- [ ] README 与 `tool_catalog` 已更新
- [ ] 相关测试补充完成
- [ ] `go test ./...` 通过

## 6. 风险与停手线

- 如果变化点摘要过长，会重新把 summary 做重
- 如果变化点归纳过强，可能掩盖原始样本细节
- 因此本版应控制在“帮助定位变化”而不是“直接下结论”

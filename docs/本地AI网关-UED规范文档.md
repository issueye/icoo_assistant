# 本地 AI 网关 UED 规范文档

## 1. 目标

本规范用于统一 `icoo_proxy` 桌面端后台管理界面的视觉风格、交互规则和基础组件实现，确保以下目标：

- 风格统一：所有页面遵循一致的后台管理视觉语言
- 交互稳定：按钮、输入、下拉、开关、表格、弹窗等组件具备一致行为
- 开发可复用：通过基础组件减少页面重复实现
- 交付可维护：后续新增页面和模块能够直接复用现有 UED 体系

## 2. 设计原则

- 传统后台优先：以 PC 管理系统为主，不走营销页或展示页风格
- 信息效率优先：强调清晰层级、表格化、表单化和状态可视化
- 中文业务优先：所有通用文案默认使用中文
- 一致性优先：颜色、间距、边框、字号、状态样式统一
- 易扩展优先：组件 API 简洁，可通过 slot 和 props 扩展

## 3. 视觉规范

### 3.1 色彩

- 主色：`#2563EB`
- 主色悬浮：`#1D4ED8`
- 成功色：`#059669`
- 警告色：`#D97706`
- 危险色：`#DC2626`
- 文本主色：`#0F172A`
- 文本次级：`#475569`
- 文本弱化：`#64748B`
- 边框色：`#CBD5E1`
- 分割线：`#E2E8F0`
- 页面背景：`#F3F6FB`
- 卡片背景：`#FFFFFF`
- 蒙层背景：`rgba(15, 23, 42, 0.45)`

### 3.2 字体

- 字体族：`Segoe UI`, `PingFang SC`, `Microsoft YaHei`, `sans-serif`
- 页面主标题：28px / 600
- 卡片标题：18px / 600
- 正文：14px / 400
- 表格正文：14px / 400
- 辅助信息：12px / 400

### 3.3 圆角与阴影

- 小圆角：8px
- 中圆角：10px
- 大圆角：12px
- 卡片阴影：轻阴影，仅用于层级区分

### 3.4 间距

- 页面块间距：20px
- 面板内边距：20px
- 表单控件纵向间距：16px
- 按钮组间距：12px
- 表格单元格内边距：12px 16px

## 4. 布局规范

### 4.1 整体布局

- 左侧固定导航栏
- 顶部信息栏
- 右侧主工作区
- 页面结构统一为：页面标题区 + 摘要区 + 内容区

### 4.2 页面头部

- 仅保留一个主标题
- 可保留一段页面说明
- 不再使用副标题、眉标题、英文抬头

### 4.3 面板区

- 每个面板只保留单一标题
- 面板内部优先使用表格、列表和表单
- 空状态统一使用虚线边框和浅灰背景

## 5. 交互规范

### 5.1 按钮

- 主按钮：核心保存、提交、确认操作
- 次按钮：返回、取消、重置、编辑等辅助操作
- 危险按钮：删除、停用、清空等高风险操作
- 文案优先使用动词，如“保存”“删除”“检查”“确认删除”

### 5.2 输入框

- 标签置于控件上方
- 默认高度统一
- 聚焦时使用主色边框和浅色外环
- 错误态需要红色边框和错误提示

### 5.3 下拉框

- 样式与输入框一致
- 默认展示当前值或“请选择”
- 禁用态要降低可视权重

### 5.4 开关

- 适合表达启用/停用
- 文案应明确说明开关影响对象

### 5.5 表格

- 表头浅灰背景
- 行 hover 提供浅色反馈
- 支持空状态
- 操作列保持右侧或末列
- 复杂内容通过插槽扩展

### 5.6 弹窗

- 居中显示
- 包含标题、内容区、底部操作区
- 支持点击关闭按钮和取消按钮
- 危险确认场景应使用确认弹窗

### 5.7 确认弹窗

- 用于删除、覆盖、关闭等高风险操作
- 必须包含明确风险说明
- 主按钮文案需体现结果，如“确认删除”

## 6. 组件规范

### 6.1 `UButton`

- 属性：`variant`、`size`、`block`、`loading`、`disabled`
- 变体：`primary`、`secondary`、`danger`、`ghost`

### 6.2 `UInput`

- 属性：`modelValue`、`label`、`placeholder`、`disabled`、`error`、`textarea`
- 行为：支持 `v-model`

### 6.3 `USelect`

- 属性：`modelValue`、`label`、`options`、`placeholder`、`disabled`、`error`
- `options` 结构：`{ label, value }`

### 6.4 `USwitch`

- 属性：`modelValue`、`label`、`hint`、`disabled`
- 行为：支持 `v-model`

### 6.5 `UTable`

- 属性：`columns`、`rows`、`rowKey`、`emptyText`
- 支持单元格插槽：`cell-字段名`
- 支持操作列插槽：`actions`

### 6.6 `UModal`

- 属性：`open`、`title`、`width`
- 事件：`update:open`、`close`
- 支持默认内容区与底部插槽

### 6.7 `UConfirmDialog`

- 属性：`open`、`title`、`message`、`confirmText`、`cancelText`、`loading`、`danger`
- 事件：`confirm`、`cancel`、`update:open`

## 7. 开发约束

- 页面优先使用 UED 基础组件，不直接拼原生表单控件
- 新增页面必须遵循本规范中的布局、文案和交互规则
- 新增组件需先补充规范，再纳入 `components/ued/`
- 业务页面中高风险操作必须接入确认弹窗

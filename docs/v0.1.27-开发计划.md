# v0.1.27 开发计划

## 1. 版本主题

让 `task_audit history` 在聚焦失败查看时补上一条 latest failure reason hint，减少用户还要从 `latest_sample` 或条目行里再抽取一次失败分类。

## 2. 为什么现在做

`v0.1.26` 已经能让用户在顶部直接看到最新失败的原始错误文本，下一步最自然的问题就是“这条最新失败在分类上到底算 timeout、command_error 还是其他类型”。这一版继续停留在 history 可读性层，不扩展到新的归因逻辑。

## 3. 计划变更

- 为 `task_audit history` 增加一个轻量 latest failure reason hint
- 让失败历史视图更容易快速对齐最新失败提示与失败分类
- README 与 `tool_catalog` 同步更新
- 增加对应测试

## 4. 本版不做

- 新的失败分类规则
- 自动解释分类置信度
- 基于失败分类的修复建议

## 5. 验收标准

- [ ] `task_audit history` 新增 latest failure reason hint
- [ ] README 与 `tool_catalog` 已更新
- [ ] 相关测试补充完成
- [ ] `go test ./...` 通过

## 6. 风险与停手线

- 如果 latest failure reason hint 引入新的分类判断，会和现有 summary 侧失败归因逻辑边界混淆
- 如果重复解释 reason 的含义，会让 history 头部再次承担诊断职责
- 因此本版应控制在“更快看到失败分类”而不是“解释分类意味着什么”

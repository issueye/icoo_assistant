# v0.1.22 开发计划

## 1. 版本主题

让 `task_audit history` 在聚焦失败查看时补上一条直接的 latest failure command 提示，减少用户继续在条目里手动寻找本次执行命令。

## 2. 为什么现在做

`v0.1.21` 已经能让用户先定位到最新失败记录，下一步最自然的问题就是“这条最新失败到底执行了什么命令”。这一版继续停留在 history 可读性层，不扩展到新的推断逻辑。

## 3. 计划变更

- 为 `task_audit history` 增加一个轻量 latest failure command 提示
- 让失败历史视图更容易快速对齐当前失败记录与执行命令
- README 与 `tool_catalog` 同步更新
- 增加对应测试

## 4. 本版不做

- 命令级 diff
- 自动命令修复建议
- 更复杂的失败分组视图

## 5. 验收标准

- [ ] `task_audit history` 新增 latest failure command 提示
- [ ] README 与 `tool_catalog` 已更新
- [ ] 相关测试补充完成
- [ ] `go test ./...` 通过

## 6. 风险与停手线

- 如果 latest failure command 提示和条目正文重复太多，会削弱 history 的紧凑性
- 如果提示开始附带命令解释，会和后续诊断层能力混在一起
- 因此本版应控制在“更快看到命令”而不是“解释命令为什么失败”

# v0.1.11 开发计划

## 1. 版本定位

`v0.1.11` 计划继续围绕失败复盘体验做小步迭代，把当前已有的优先依据说明，再推进到更适合快速看到“这一类失败最近具体怎么表现”的方向。

## 2. 版本目标

本版本计划交付以下能力：

- 增强优先失败原因的上下文摘要
- 优化从优先依据到详细历史的路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.11`

- 异常复盘相关审计增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.11`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 优先提升失败原因摘要的可读性
- 不破坏既有工具边界

### 4.2 版本重点

- 优先失败原因的上下文摘要
- 失败复盘路径的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### A. 异常复盘

- 评估是否需要补充优先失败原因的最近样本摘要或错误上下文
- 优化从优先依据到失败历史查看的路径

### B. 演示整理

- 增加更贴近优先失败原因摘要的示例
- 保持 README 简洁可执行

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [x] 更容易理解优先失败原因最近的表现
- [x] 失败复盘路径更顺滑
- [x] 测试通过
- [x] 文档更新

## 7. 完成度评估

### 7.1 当前结论

`v0.1.11` 可以判定为已完成，当前任务侧审计已经能在优先排查建议旁边直接给出最近一条同类失败样本的上下文摘要，减少从概况跳到详细历史前的判断成本。

完成依据：

- 已在 `task_audit summary` 中新增 `priority_failure_context`
- 优先失败原因现在不仅说明为何被优先，还会展示最近一条同类失败的 `job_id / command / error` 摘要
- README 与 `tool_catalog` 已同步补上优先失败样本摘要的示例与导航
- 相关测试已补充并通过 `go test ./...`

### 7.2 当前边界

- 当前上下文摘要仍然只展示最近一条样本，不做跨样本归纳
- 仍然是单任务视角，暂不支持跨任务优先失败样本对比
- 还没有进一步把错误文本压缩成更高层标签或修复建议

## 8. 风险提示

- 如果上下文摘要继续增加，可能压缩掉其他关键信息
- 若摘要选择不稳定，可能影响对比体验
- 仍需持续控制范围膨胀

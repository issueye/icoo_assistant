# v0.0.9 开发计划

## 1. 版本定位

`v0.0.9` 聚焦把当前任务历史查看入口进一步推进到“更接近审计视角”的体验。核心目标是为后续独立审计能力铺路，同时避免让主任务工具承担过多历史查询职责。

## 2. 版本目标

本版本计划交付以下能力：

- 更独立的历史查询动作或工具方向
- 更稳定的历史输出格式
- 相关测试和文档

## 3. 范围定义

### 3.1 纳入 `v0.0.9`

- 历史查询入口增强
- 输出格式进一步规范
- 文档与测试

### 3.2 暂不纳入 `v0.0.9`

- 自动任务拆解
- 自动认领任务
- 多 Agent 团队协作
- 审批协议
- worktree 绑定

## 4. 设计重点

### 4.1 核心原则

- 审计能力要逐步独立
- 默认任务操作入口继续保持简洁
- 输出格式要便于后续脚本化或导出

### 4.2 版本重点

- 先规范查询体验
- 再考虑独立审计存储

## 5. 任务拆解

### A. 查询入口

- 评估是否需要进一步拆分历史查询动作
- 统一入参和输出约定

### B. 输出规则

- 优化历史输出结构
- 兼顾人类可读和后续机器处理

### C. 测试与文档

- 增加查询格式测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [ ] 历史查询体验进一步增强
- [ ] 输出格式更稳定
- [ ] 测试通过
- [ ] 文档更新

## 7. 风险提示

- 如果入口拆得太散，工具体系会变复杂
- 如果输出格式改动过大，可能影响现有 prompt 使用习惯
- 审计能力设计要为后续独立存储保留余地

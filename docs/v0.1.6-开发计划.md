# v0.1.6 开发计划

## 1. 版本定位

`v0.1.6` 计划继续围绕失败排障体验做小步迭代，把当前已有的失败概况和原因分类，再推进到更适合快速比对和复盘的方向。

## 2. 版本目标

本版本计划交付以下能力：

- 增强失败信息的对比或聚焦体验
- 优化从失败概况到详细复盘的路径
- 继续补齐文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.6`

- 异常排障相关审计增强
- README 示例补强
- 文档与测试

### 3.2 暂不纳入 `v0.1.6`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 可视化界面

## 4. 设计重点

### 4.1 核心原则

- 保持小步迭代
- 优先提升真实失败排障场景的效率
- 不破坏既有工具边界

### 4.2 版本重点

- 失败模式的可比较性
- 失败复盘路径的顺滑度
- 文档与代码一致性

## 5. 任务拆解

### A. 异常排障

- 评估是否需要更细的失败聚焦、对比或排序能力
- 优化从失败概况到详细复盘的路径

### B. 演示整理

- 增加更贴近失败复盘的示例
- 保持 README 简洁可执行

### C. 测试与发布

- 增加相关测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [x] 失败模式更容易被对比和复盘
- [x] 失败排障路径更顺滑
- [x] 测试通过
- [x] 文档更新

## 7. 完成度评估

### 7.1 当前结论

`v0.1.6` 可以判定为已完成，当前任务侧审计已经能在失败概况里按原因分类展示最近样本，失败模式更容易被对比和复盘。

完成依据：

- 已在 `task_audit summary` 中新增 `latest_failure_by_reason`
- 失败原因分类现在不仅有计数，还能看到每类失败的最近样本
- README 与 `tool_catalog` 已同步补上失败样本对比导向的示例与说明
- 相关测试已补充并通过 `go test ./...`

### 7.2 当前边界

- 当前失败样本对比仍然只基于最近一条，不会展示同类失败的更长趋势
- 失败原因分类依旧依赖基础文本规则，尚未形成可扩展的分类体系
- 仍然是单任务视角，跨任务复盘还没有纳入

## 8. 风险提示

- 如果失败聚焦能力继续增加，需要避免输出约定变复杂
- 若对比信息过多，可能削弱快速浏览体验
- 仍需持续控制范围膨胀

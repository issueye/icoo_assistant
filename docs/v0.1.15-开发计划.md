# v0.1.15 开发计划

## 1. 版本主题

让 `task_audit summary` 不只展示最近样本差异，还能直接提示这类失败最近是趋于稳定还是正在变化。

## 2. 为什么现在做

`v0.1.14` 已经能对比优先失败原因最近两条样本，但用户还需要自己再解释“这是稳定重复还是开始漂移”。这一版只补一个轻量趋势字段，不直接做更重的跨任务趋势或修复建议。

## 3. 计划变更

- 在 `task_audit summary` 中新增 `priority_failure_trend_hint`
- 基于优先失败原因最近样本输出 `emerging / stable / changing`
- README 增加趋势提示示例
- `tool_catalog` 补充趋势提示导航
- 增加对应测试

## 4. 本版不做

- 跨任务趋势汇总
- 基于趋势的自动修复建议
- 更长窗口的统计图或可视化

## 5. 验收标准

- [x] `task_audit summary` 新增 `priority_failure_trend_hint`
- [x] 测试覆盖 `emerging / stable / changing` 三类输出
- [x] README 与 `tool_catalog` 已更新
- [x] `go test ./...` 通过

## 6. 完成度评估

### 6.1 当前结论

`v0.1.15` 可以判定为已完成，当前任务侧审计已经能直接提示优先失败原因最近是新近出现、稳定重复，还是已经开始变化。

完成依据：

- 已在 `task_audit summary` 中新增 `priority_failure_trend_hint`
- 当前输出会基于最近样本给出 `emerging / stable / changing`
- README 与 `tool_catalog` 已同步补上趋势提示示例与导航
- 相关测试已补充并通过 `go test ./...`

### 6.2 当前边界

- 当前趋势仍然只基于最近少量样本
- 趋势判断仍然依赖轻量规则，不包含更高层原因分析
- 仍然没有扩展到跨任务或跨时间窗口的统计

## 7. 风险与停手线

- 如果继续叠加摘要字段，首屏可读性会继续下降
- 如果趋势解释开始带推断色彩，可能误导排障优先级
- 因此这一版停在轻量趋势提示，不继续扩展到修复建议

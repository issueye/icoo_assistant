# v0.1.26 开发计划

## 1. 版本主题

让 `task_audit history` 在聚焦失败查看时补上一条 latest failure error hint，减少用户还要下翻条目列表确认最新失败原始错误文案。

## 2. 为什么现在做

`v0.1.25` 已经能让用户更快把最新失败提示对到具体条目，下一步最自然的问题就是“这条最新失败在顶部能不能直接看到原始错误文本”。这一版继续停留在 history 可读性层，不扩展到新的诊断逻辑。

## 3. 计划变更

- 为 `task_audit history` 增加一个轻量 latest failure error hint
- 让失败历史视图更容易快速对齐最新失败提示与原始错误文本
- README 与 `tool_catalog` 同步更新
- 增加对应测试

## 4. 本版不做

- 自动提炼更长的错误摘要
- 新的错误归因逻辑
- 基于错误文本的修复建议

## 5. 验收标准

- [ ] `task_audit history` 新增 latest failure error hint
- [ ] README 与 `tool_catalog` 已更新
- [ ] 相关测试补充完成
- [ ] `go test ./...` 通过

## 6. 风险与停手线

- 如果 latest failure error hint 开始承载摘要提炼，会和现有 `latest_failure_signature` 混淆
- 如果错误文本过长且不做边界控制，可能削弱 history 头部的紧凑性
- 因此本版应控制在“更快看到原始错误文本”而不是“解释错误意味着什么”

# v0.1.1 开发计划

## 1. 版本定位

`v0.1.1` 计划承接已经落盘的 Agent hook 埋点，把“能记录”继续推进到“能查询、能回看、能用于排障”的阶段。

## 2. 版本目标

本版本计划交付以下能力：

- 增加基础 hook 查询或审计入口
- 让最近一次运行的关键事件更容易回看
- 补齐对应文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.1`

- `.agent-hooks/events.jsonl` 基础查询能力
- 最近事件或过滤能力
- 文档与测试

### 3.2 暂不纳入 `v0.1.1`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 完整可视化界面

## 4. 设计重点

### 4.1 核心原则

- 先把现有 hook 数据利用起来
- 优先支持演示、排障和运行回看
- 尽量复用现有文件落盘结构

### 4.2 版本重点

- hook 查询体验
- 事件输出可读性
- 与当前运行时说明保持一致

## 5. 任务拆解

### A. Hook 查询

- 设计基础 hook 查询工具或动作
- 支持按最近条数或事件类型过滤

### B. 输出整理

- 统一 hook 查询输出格式
- 明确与任务审计的边界

### C. 测试与发布

- 增加 hook 查询测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [x] 可以查询最近 hook 事件
- [x] hook 查询和任务审计边界清晰
- [x] 测试通过
- [x] 文档更新

## 7. 完成度评估

### 7.1 当前结论

`v0.1.1` 可以判定为已完成，当前运行时已经具备独立的 Agent hook 审计查询能力，能够直接回看最近运行事件，并且和项目任务审计保持清晰分工。

完成依据：

- 已新增 `agent_hook_audit` 工具，支持 `recent`
- 已支持通过 `limit / name / run_id` 查询最近 hook 事件
- 已补充 `internal/hookaudit` 的 hook 读取能力，复用现有 `.agent-hooks/events.jsonl`
- 已将 `agent_hook_audit` 接入主运行时和子代理基础工具集合
- README 与工具边界说明已同步更新
- `go test ./...` 通过

### 7.2 当前边界

- hook 查询仍然基于本地 JSONL 顺序扫描，后续事件量继续增长时可能需要更高效索引
- 当前输出以文本审计视图为主，还没有结构化导出接口
- 目前只支持最近事件回看，还没有按时间范围或字段键值做更细粒度过滤

## 8. 风险提示

- hook 事件量增长后，读取策略需要控制开销
- 查询输出过长会影响交互体验
- 需要避免和 `task_audit` 的职责重叠

# v0.1.1 开发计划

## 1. 版本定位

`v0.1.1` 计划承接已经落盘的 Agent hook 埋点，把“能记录”继续推进到“能查询、能回看、能用于排障”的阶段。

## 2. 版本目标

本版本计划交付以下能力：

- 增加基础 hook 查询或审计入口
- 让最近一次运行的关键事件更容易回看
- 补齐对应文档与测试

## 3. 范围定义

### 3.1 纳入 `v0.1.1`

- `.agent-hooks/events.jsonl` 基础查询能力
- 最近事件或过滤能力
- 文档与测试

### 3.2 暂不纳入 `v0.1.1`

- 多 Agent 团队协作
- 审批协议
- 自动任务拆解
- 自动认领任务
- 完整可视化界面

## 4. 设计重点

### 4.1 核心原则

- 先把现有 hook 数据利用起来
- 优先支持演示、排障和运行回看
- 尽量复用现有文件落盘结构

### 4.2 版本重点

- hook 查询体验
- 事件输出可读性
- 与当前运行时说明保持一致

## 5. 任务拆解

### A. Hook 查询

- 设计基础 hook 查询工具或动作
- 支持按最近条数或事件类型过滤

### B. 输出整理

- 统一 hook 查询输出格式
- 明确与任务审计的边界

### C. 测试与发布

- 增加 hook 查询测试
- 更新 README
- 更新版本文档

## 6. 验收清单

- [ ] 可以查询最近 hook 事件
- [ ] hook 查询和任务审计边界清晰
- [ ] 测试通过
- [ ] 文档更新

## 7. 风险提示

- hook 事件量增长后，读取策略需要控制开销
- 查询输出过长会影响交互体验
- 需要避免和 `task_audit` 的职责重叠

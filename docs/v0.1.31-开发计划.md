# v0.1.31 开发计划

## 1. 版本主题

在 `v0.1.30` 初代可用版之后，补一轮 post-release hardening，优先收集并修正最小闭环里仍然容易产生误解的文案或交互细节。

## 2. 为什么现在做

`v0.1.30` 已经完成初代可用版交付收口，下一步不应立刻打开新的大功能，而应先围绕真实上手路径做一轮轻量稳定化，降低初次使用时的残余摩擦。

## 3. 计划变更

- 复盘 `assistant check`、help、README 和最小 happy path 中仍然容易误解的点
- 优先修正文案或交互层的小摩擦
- 补充对应测试或 smoke 验证
- 保持版本粒度轻量

## 4. 本版不做

- 全新工具主题
- 大规模架构重构
- 发布渠道扩展

## 5. 验收标准

- [ ] 至少一个初代可用版遗留摩擦点已被修正
- [ ] 相关测试或 smoke 验证补充完成
- [ ] README 或 CLI 文案已同步更新
- [ ] `go test ./...` 通过

## 6. 风险与停手线

- 如果立刻重启大功能开发，会打断 `v0.1.30` 刚建立起来的可交付基线
- 如果 post-release hardening 没有真实问题导向，又容易退回机械小修小补
- 因此本版应控制在“围绕真实上手摩擦做针对性修正”而不是“重新发散主题”

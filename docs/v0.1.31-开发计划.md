# v0.1.31 开发计划

## 1. 版本主题

在 `v0.1.30` 初代可用版之后，补一轮 post-release hardening，优先收集并修正最小闭环里仍然容易产生误解的文案或交互细节。

## 2. 为什么现在做

`v0.1.30` 已经完成初代可用版交付收口，下一步不应立刻打开新的大功能，而应先围绕真实上手路径做一轮轻量稳定化，降低初次使用时的残余摩擦。

## 3. 计划变更

- 统一 `assistant check`、`--help` 与 README 中的首次上手路径编号
- 在 `assistant check` 中明确提示当前已经完成第 1 步，应从第 2 步继续
- 在 `--help` 中补齐 `doctor` 别名和 fake mode dry run 说明
- 补充对应测试或 smoke 验证
- 保持版本粒度轻量

## 4. 本版不做

- 全新工具主题
- 大规模架构重构
- 发布渠道扩展

## 5. 验收标准

- [x] 至少一个初代可用版遗留摩擦点已被修正
- [x] 相关测试或 smoke 验证补充完成
- [x] README 或 CLI 文案已同步更新
- [x] `go test ./...` 通过

## 6. 完成度评估

`v0.1.31` 可以判定为已完成，本版针对首次上手路径里的一个真实遗留摩擦点做了收口：此前 README、`assistant check` 与 `--help` 虽然都在讲最小 happy path，但编号和“下一步从哪里继续”没有完全对齐，用户第一次执行 `assistant check` 后仍需要自己判断接下来该跑哪一条命令。

本版完成情况如下：

- `assistant check` 现在会输出完整的 1 到 4 步首次上手路径
- `assistant check` 会明确标注“第 1 步 assistant check 已完成”，并提示从第 2 步继续
- `assistant --help` 现在和 README 使用同一套首次上手路径
- `assistant --help` 补充了 `doctor` 别名和 fake mode dry run 说明
- 对应 CLI 测试已补齐，`go test ./...` 已通过

当前边界仍保持不变：

- 本版没有开启新大功能主题
- 本版仍属于 `v0.1.30` 之后的 post-release hardening
- 下一版应继续评估是否还有高频上手摩擦；如果没有，就应切向 Team 主线

## 7. 风险与停手线

- 如果立刻重启大功能开发，会打断 `v0.1.30` 刚建立起来的可交付基线
- 如果继续在 README、help、自检文案上做低价值微调，又会退回机械小修小补
- 因此本版停在“统一首次上手路径与下一步提示”，不继续扩展成更复杂的新手引导系统

# v0.1.21 开发计划

## 1. 版本主题

让 `task_audit history` 在聚焦失败查看时补上更直接的 latest sample 提示，减少用户自己判断当前最值得先看的那条记录。

## 2. 为什么现在做

`v0.1.20` 已经让 history 条目里能直接看到 failure reason，下一步最自然的问题就是“当前应该先盯哪条最新失败”。这一版继续停留在历史可读性层，不扩展到新的分析逻辑。

## 3. 计划变更

- 为 `task_audit history` 增加一个轻量 latest sample 提示
- 让失败历史视图更容易快速定位当前最新失败记录
- README 与 `tool_catalog` 同步更新
- 增加对应测试

## 4. 本版不做

- 自动推荐修复动作
- 新的失败原因分类
- 多任务联合视图

## 5. 验收标准

- [ ] `task_audit history` 新增 latest sample 提示
- [ ] README 与 `tool_catalog` 已更新
- [ ] 相关测试补充完成
- [ ] `go test ./...` 通过

## 6. 风险与停手线

- 如果 latest sample 提示重复已有条目内容过多，会削弱 history 的紧凑性
- 如果提示开始承载解释语义，会和 summary 的导航字段重叠
- 因此本版应控制在“更快定位最新记录”而不是“追加新的判断层”
